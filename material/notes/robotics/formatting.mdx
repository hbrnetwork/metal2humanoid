This page exercises every markdown, MDX, and custom-plugin feature available in the site's
content pipeline. It is **not linked from navigation, not in the sitemap, blocked in
robots.txt**, and is intended as a live visual reference paired with
[src/content/README.md](../../content/README.md).

> If something on this page looks broken, the corresponding plugin or stylesheet is
> likely broken for real notes too.

## 1. Headings & automatic anchors

## 5. Callouts (custom — `remark-callouts`)

Callouts use the Obsidian/GitHub-style `> [!type]` syntax.

> [!note] Note callout
> Blue theme — informational content. Also `info`, `todo`.

> [!tip] Tip callout
> Teal theme — helpful hint. Also `hint`.

> [!abstract] Abstract / Summary
> Teal theme. Alias: `summary`.

> [!success] Success
> Green theme. Alias: `check`.

> [!question] Question
> Amber theme. Also `help`.

> [!warning] Warning
> Amber theme. Also `caution`, `attention`.

> [!failure] Failure
> Red theme. Aliases: `fail`, `danger`, `error`, `bug`.

> [!important] Important
> Coral theme — stands out more than warning.

> [!example] Example
> Purple theme.

> [!quote] Quote
> Gray theme. Alias: `cite`.

> [!math] Mathematical callout
> Purple theme — reserved for definitions / statements of theorems.

> [!derivation] Derivation — foldable by default (collapsed)
> Click the title to expand. `derivation` and `proof` default to foldable.

> [!proof] Proof
> Also foldable by default.

### 5.1 Foldable callouts (explicit control)

> [!tip]- Foldable, collapsed by default
> The `-` suffix means collapsed initially.

> [!tip]+ Foldable, expanded by default
> The `+` suffix means expanded initially.

### 5.2 Fold separator (`---` inside a callout)

> [!note]- Callout with a fold separator
> This part is **always visible**, even when the callout is collapsed.
> ---
> This part is hidden until the user expands the callout. Useful for a short
> summary followed by detailed content.

## 6. Wiki-links (custom — `remark-wiki-links`)

Wiki-links always resolve to `/notes/{slug}/`. The target is slugified
automatically.

- Bare link: [[transformers-attention]]
- With display text: [[transformers-attention|the transformer paper note]]
- With heading anchor: [[transformers-attention#self-attention]]
- Anchor + display text: [[transformers-attention#self-attention|jump directly to self-attention]]

On real notes, hovering a wiki-link highlights the matching node in the
knowledge graph panel.

### 6.1 Links to notes that don't exist yet

If a wiki-link points to a note that doesn't exist, it is marked with a
dotted underline and a 🚧 icon. Clicking it opens a modal explaining that
the note is under construction, rather than a 404 page. Try: [[lie-groups-introduction|Lie groups (under construction)]] or
[[reinforcement-learning-introduction|reinforcement learning]]. (On this
sandbox page the modal only mounts when the reader layout is used — see
`/notes/` pages for the live behaviour.)

## 7. Citations (custom — `remark-citations`)

Citations use `[@key]` and are rendered as superscript numeric links that
point to the auto-generated References section at the bottom.

Single citation: This paper [@vaswani2017] introduced the transformer.

Multiple in one bracket: Attention mechanisms originate earlier [@bahdanau2015; @luong2015],
and were later unified in [@vaswani2017].

The citation keys resolve to the `references` frontmatter block at the top of
this file.

## 8. Math (KaTeX)

Inline math uses single dollar signs: $e^{i\pi} + 1 = 0$.

Display math uses double dollar signs on their own lines:

$$
\text{Attention}(Q, K, V) = \text{softmax}\!\left(\frac{QK^\top}{\sqrt{d_k}}\right) V
$$

Full LaTeX environments are supported:

$$
\begin{aligned}
\nabla \cdot \mathbf{E} &= \frac{\rho}{\varepsilon_0} \\
\nabla \cdot \mathbf{B} &= 0 \\
\nabla \times \mathbf{E} &= -\frac{\partial \mathbf{B}}{\partial t} \\
\nabla \times \mathbf{B} &= \mu_0 \mathbf{J} + \mu_0 \varepsilon_0 \frac{\partial \mathbf{E}}{\partial t}
\end{aligned}
$$

Cases and matrices:

$$
f(x) = \begin{cases} x^2 & \text{if } x \geq 0 \\ -x & \text{otherwise} \end{cases}
\qquad
A = \begin{bmatrix} 1 & 2 \\ 3 & 4 \end{bmatrix}
$$

Clicking any rendered formula silently copies its LaTeX source to the
clipboard (handled by `mount-reader.jsx` on real note pages).

## 9. Code blocks (`rehype-pretty-code` + Shiki)

Plain fenced block:

```
plain text, no highlighting
```

With language:

```python
def hello():
    print("world")
```

With `title="…"` (filename badge) and line highlighting with `{4-5,7}`:

```python title="attention.py" {4-5,7}
import torch
import torch.nn.functional as F

def scaled_dot_product_attention(Q, K, V, mask=None):
    d_k = Q.size(-1)
    scores = torch.matmul(Q, K.transpose(-2, -1)) / (d_k ** 0.5)
    if mask is not None:
        scores = scores.masked_fill(mask == 0, float('-inf'))
    weights = F.softmax(scores, dim=-1)
    return torch.matmul(weights, V), weights
```

Multiple languages work out of the box (rust, go, js, ts, json, yaml, bash, …):

```rust
fn fib(n: u32) -> u32 {
    if n < 2 { n } else { fib(n - 1) + fib(n - 2) }
}
```

```bash
npm run dev          # start Astro dev server
npm run build        # full production build
npm run graph        # rebuild public/graph.json only
```

## 13. Raw HTML & inline styles

Because this is MDX, any JSX / HTML element is valid. For a tinted info box
that isn't a callout:

<div style="padding: 1rem; background: color-mix(in srgb, var(--accent) 10%, transparent); border-left: 3px solid var(--accent); border-radius: 4px; margin: 1rem 0;">
  <strong>Inline styled box.</strong> Useful when callouts don't fit — e.g.,
  quick visual treatments without a title or icon.
</div>

## 14. Importing and using React components

At the top of an MDX file:

```jsx
import GlobeSatellites from '../../components/GlobeSatellites.jsx';
```

Then use it anywhere in the body. Client-only components (ones that rely on
`window`, `document`, or a browser API) must be marked with
`client:only="react"`:

```jsx
<GlobeSatellites client:only="react" />
```

Other directives available: `client:load`, `client:idle`, `client:visible`,
`client:media`. Prefer `client:visible` for expensive components below the
fold.
